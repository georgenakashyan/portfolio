# External Integrations

**Analysis Date:** 2026-01-16

## APIs & External Services

**Form Submission:**
- Formspree - Contact form backend
  - Endpoint: `https://formspree.io/f/${NEXT_PUBLIC_FORMSPREE_ID}`
  - Auth: Public form ID in environment variable
  - Implementation: `app/components/sections/ContactForm.tsx`
  - Method: POST with JSON body (name, email, message)

**Analytics:**
- Vercel Analytics - Page view and performance tracking
  - SDK: `@vercel/analytics` ^1.6.1
  - Import: `import { Analytics } from "@vercel/analytics/next"`
  - Implementation: `app/layout.tsx` - Analytics component in main layout
  - Auth: Automatic via Vercel deployment (no env var needed)

## Data Storage

**Databases:**
- None - This is a static portfolio site
- Content stored in filesystem (`content/projects/*.mdx`)
- Project metadata in TypeScript constants (`lib/constants/projects.ts`)

**File Storage:**
- Local filesystem only
- Static assets in `public/` directory
  - `public/George-Nakashyan-Resume.pdf`
  - `public/Prisoners_Dilemma_Simulation.pdf`
  - `public/assets/`
  - `public/images/`

**Caching:**
- Next.js built-in caching (ISR, static generation)
- No external cache service

## Authentication & Identity

**Auth Provider:**
- None - Public portfolio site with no user authentication
- Contact form uses Formspree for spam protection

## Monitoring & Observability

**Error Tracking:**
- None configured
- Console logging only in ContactForm error handler

**Logs:**
- Browser console for client-side errors
- No server-side logging service

**Analytics:**
- Vercel Analytics for page views and web vitals
- No custom event tracking implemented

## CI/CD & Deployment

**Hosting:**
- Vercel
- Config: `vercel.json` with URL rewrite rules
- Auto-deployment from Git (standard Vercel workflow)

**CI Pipeline:**
- Vercel's built-in CI (build on push)
- No custom GitHub Actions or other CI

**Build:**
- `npm run build` (Next.js production build)
- Output: Hybrid (static + server-rendered)

## Environment Configuration

**Required env vars:**
```bash
NEXT_PUBLIC_SITE_URL=https://georgenakashyan.com  # SEO base URL
NEXT_PUBLIC_MY_EMAIL=georgenakashyan@gmail.com    # Contact display
NEXT_PUBLIC_FORMSPREE_ID=<form-id>                # Formspree endpoint
```

**Secrets location:**
- `.env` file (local development)
- Vercel Environment Variables (production)
- All variables are `NEXT_PUBLIC_*` (client-safe, no secrets)

**Default fallbacks:**
- Site URL defaults to `https://georgenakashyan.com`
- Email defaults to `georgenakashyan@gmail.com`

## Webhooks & Callbacks

**Outgoing:**
